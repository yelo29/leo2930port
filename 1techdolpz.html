Introduction to HTML

HTML (HyperText Markup Language) is the standard language for creating web pages. It provides the structure of a page using elements such as headings, paragraphs, lists, links, and images.

A basic HTML element looks like: <p>Your text here</p>. The doctype declaration at the top tells the browser which HTML version to expect.

  • Elements: <h1> - <h6>, <p>, <a>, <img>
  • Attributes: provide extra information, e.g. src and alt for images

Example block: 

<!doctype html>
<html lang="en">
  <head><meta charset="utf-8"></head>
  <body>
    <p>Hello world!</p>
  </body>
</html>
CSS Basics

CSS (Cascading Style Sheets) controls the presentation of HTML. Rules (selectors + declarations) define how elements appear.

Example of a CSS rule: p { color: #333; line-height: 1.6; }. Classes and IDs allow targeted styling.

  • Selectors: element, class .example, id #example
  • Box model: margin, border, padding, content
  • Specificity: determines which rule wins when multiple rules apply

Use an external stylesheet by linking with: <link rel="stylesheet" href="styles.css">.

JavaScript and DOM

JavaScript adds interactivity. Script code runs in the browser and can manipulate the DOM (Document Object Model).

Example: selecting an element and changing its text: document.querySelector('h1').textContent = 'Updated';

The DOM represents HTML elements as objects. You can create, remove, or update elements dynamically.

  • Event listeners: element.addEventListener('click', callback)
  • Creating elements: document.createElement('div')
// Simple click handler
const btn = document.getElementById('myBtn');
btn.addEventListener('click', () => {
  alert('Button clicked!');
});
Responsive Design

Responsive design ensures web pages look and function well on a variety of devices and screen sizes.

Use flexible grids, fluid images, and media queries. Example: @media (max-width: 800px) { ... }.

  • Fluid grids: use relative units like %, rem, or em
  • Flexible images: img { max-width: 100%; height: auto; }
/* Example media query */
@media (max-width: 600px) {
  .sidebar { display: none; }
  .content { margin-left: 0; }
}

Progressive enhancement and mobile-first CSS help keep interfaces simple and performant.

Web Accessibility

Accessibility (a11y) makes content usable by people with disabilities. Follow semantic HTML, use ARIA where necessary, and ensure keyboard navigation.

Provide descriptive alt text for images, label form fields, and ensure sufficient color contrast.

  • Keyboard focus order and visible focus styles
  • Semantic elements: <nav>, <main>, <header>, <footer>

Example: use <nav aria-label="Primary"> to help screen readers find navigation. 

<button aria-pressed="false">Toggle</button>
